# 专属教材单元上传器 (Course Unit Uploader)

## 🎯 核心功能与路由优先级 (Core Function & Routing)
这是一个高优先级的自动化上传工具。
**【绝对拦截指令】**：只要用户的对话中提到了具体的**“教材名称”**、**“项目名称”**（例如：CBBC、2026、英语启蒙等），或者明确要求**“新建单元”**，你必须**绝对优先**调用本技能！绝对不能调用旧版的普通批量上传技能。

## 🧠 AI 行为准则与参数提取 (AI Behavioral Directives)
你需要从用户的自然语言中提取两个核心参数来运行 `main.py`：
1. `-c` 或 `--course` **(必填)**：目标教材的名字。
2. `-f` 或 `--folder` **(选填)**：要上传的本地文件夹名字。如果不填，代表上传所有文件夹。

**🚨 缺失参数的反问机制 (CRITICAL)：**
如果你判断用户想把视频传到某个教材，但**你没有在对话中找到要传哪一个具体的文件夹**（例如用户只说了：“帮我把视频传到CBBC 2026”）：
**不要直接执行脚本！** 你必须像真人助理一样，先主动反问用户：“好的，请问您想上传本地的哪一个文件夹？还是要把所有文件夹都传上去？” 得到明确答复后，再拼接参数执行。

## 🗣️ 口语翻译示例 (Natural Language Translation Examples)
以下是用户可能会说的口语，以及你应该如何提取参数执行：

* **口语：** “请帮我将【4月】文件夹中的视频上传到【CBBC 2026】中”
    **执行：** `python main.py -c "CBBC 2026" -f "4月"`
* **口语：** “把【测试动画】作为新单元传到后台的【英语启蒙 第一季】教材里”
    **执行：** `python main.py -c "英语启蒙 第一季" -f "测试动画"`
* **口语：** “把本地所有的视频文件夹，全部批量上传到【默认测试教材】中”
    **执行：** `python main.py -c "默认测试教材"`

## 💻 运行环境 (Requirements)
* 依赖环境：`pip install -r requirements.txt` 和 `playwright install chromium`。
* 依赖文件：必须确保目录下存在有效的 `auth.json` 登录凭证。

## ✅ 预期结果 (Expected Result)
脚本将自动搜索指定教材，以文件夹名称新建单元，并上传其中的 `.mp4/.avi/.mkv` 文件，最后自动点击暂存。
